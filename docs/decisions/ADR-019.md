# ADR-019 - Core to AI Engine REST Submission Contract

## Status

Accepted

## Date

2026-07-21

## Context

The Java Core orchestrates the deterministic analysis workflow and owns the lifecycle of `AiTask`.

The Python AI Engine is responsible for AI interpretation but does not own the business workflow.

Communication between both services follows the asynchronous architecture established in ADR-010.

A stable and versionable contract is required between the two services.

The contract must:

* decouple Java from Python implementation details;
* support asynchronous processing;
* preserve traceability;
* remain evolvable without affecting the Core domain model.

## Decision

The Core communicates with the AI Engine using REST.

Communication consists of two independent operations:

```text
Core
   │
   │ POST /api/v1/ai/tasks
   ▼
AI Engine
   │
   │ 202 Accepted
   ▼
Core

...

AI Engine
   │
   │ POST /api/v1/ai/tasks/{correlationId}/result
   ▼
Core
```

The first operation only acknowledges task acceptance.

The second operation delivers the completed AI result asynchronously.

## Submission Request

The Core sends an immutable task request.

Conceptually:

```text
AiTaskSubmissionRequest
├── correlationId
├── taskType
├── analysisId
└── context
```

The request contains the complete deterministic context prepared by the Core.

The AI Engine must not retrieve additional project knowledge from the Core.

## Submission Response

The AI Engine immediately acknowledges reception.

```text
AiTaskSubmissionResponse
├── correlationId
├── accepted
├── externalJobId
└── acceptedAt
```

The response confirms only that processing has been accepted.

It does not contain AI-generated content.

## State Transition

Successful submission changes:

```text
CREATED
    │
    ▼
SUBMITTED
```

Submission failure changes:

```text
CREATED
    │
    ▼
FAILED
```

Processing state is reported later by the AI Engine.

## AIEngineClient

The Core communicates through an abstraction:

```text
AIEngineClient
        │
        ▼
RestAIEngineClient
```

The workflow service depends only on `AIEngineClient`.

HTTP implementation details remain isolated inside the REST client.

## Responsibilities

### Java Core

Responsible for:

* building the deterministic context;
* creating the task;
* generating the correlation identifier;
* submitting the request;
* handling transport failures;
* updating task status.

### Python AI Engine

Responsible for:

* accepting the task;
* executing AI processing;
* maintaining its internal execution state;
* sending the asynchronous result.

## Alternatives Considered

### Synchronous AI execution

Rejected because AI processing may take several seconds or minutes.

### Direct database sharing

Rejected because it tightly couples the Core and AI Engine.

### Message broker

Rejected for V1 in accordance with ADR-010.

## Consequences

### Positive

* Clear separation of responsibilities.
* Stable inter-service contract.
* Evolvable independently.
* Easy to mock and test.
* Compatible with future retries.

### Negative

* Requires callback implementation.
* Requires explicit failure handling.
* Adds a network boundary between services.
