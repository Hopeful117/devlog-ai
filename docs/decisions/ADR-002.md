# ADR-002 - Knowledge First Architecture

## Status

Accepted

## Date

2026-07-20

## Context

Traditional software documentation often becomes outdated because it is manually maintained and disconnected from the real evolution of a project.

A simple documentation generator would only transform existing information into a different format without solving the underlying problem of lost technical context.

The main objective of DevLog AI is to preserve the evolution, decisions, and reasoning behind software projects.

## Decision

DevLog AI will follow a knowledge-first architecture.

The system will maintain a structured representation of project knowledge as the primary source of truth.

Documentation outputs are considered projections generated from validated knowledge.

The knowledge lifecycle follows:

Project Activity

↓

Raw Knowledge

↓

Deterministic Analysis

↓

AI Interpretation

↓

Human Validation

↓

Project Knowledge

↓

Documentation Outputs

## Alternatives Considered

### Documentation-first approach

Generate articles and documentation directly from repository activity.

Rejected because it focuses on outputs without preserving reusable project understanding.

### Git history as the source of truth

Use commits and repository history as the complete project memory.

Rejected because Git explains what changed but rarely explains why decisions were made.

### AI-generated summaries without structured knowledge

Generate summaries directly using AI.

Rejected because generated content would lack traceability and reliability.

## Consequences

### Positive

- Documentation remains consistent with project knowledge.
- Multiple outputs can be generated from the same knowledge base.
- Technical decisions and challenges are preserved.
- The system can provide contextual assistance about projects.

### Negative

- Requires designing and maintaining a knowledge model.
- Initial development complexity is higher.
- Human validation workflow is required.
