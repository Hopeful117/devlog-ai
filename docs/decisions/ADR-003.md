# ADR-003 - Hybrid AI Architecture

## Status

Accepted

## Date

2026-07-20

## Context

DevLog AI combines software engineering analysis with artificial intelligence capabilities.

Repository analysis requires both reliable technical understanding and contextual interpretation.

Pure deterministic analysis is insufficient because understanding engineering intent requires reasoning.

Pure AI-driven analysis is also insufficient because generated conclusions may lack reliability, traceability, and consistency.

The system therefore needs a clear separation between objective project facts and AI interpretation.

## Decision

DevLog AI will use a hybrid architecture separating deterministic processing from AI capabilities.

The Java Core Service is responsible for objective project knowledge.

Responsibilities include:

- project state management,
- repository analysis coordination,
- knowledge storage,
- domain rules,
- validation workflows,
- maintaining the source of truth.

The Python AI Service is responsible for interpretation and intelligence capabilities.

Responsibilities include:

- semantic analysis,
- event and decision proposals,
- contextual understanding,
- content generation,
- AI-assisted reasoning.

The AI Service consumes structured context provided by the Core Service and does not directly own project knowledge.

## Alternatives Considered

### Full AI-driven architecture

Use AI models to directly analyze repositories and maintain project knowledge.

Rejected because:

- AI output can be inconsistent,
- reasoning may not be traceable,
- business data ownership becomes unclear.

### Fully deterministic architecture

Implement all analysis using predefined rules.

Rejected because:

- understanding engineering intent requires interpretation,
- technical decisions cannot always be inferred from code changes alone.

### AI as the central system of record

Allow the AI layer to store and manage project knowledge directly.

Rejected because:

- knowledge lifecycle becomes dependent on AI behavior,
- validation and consistency become harder to guarantee.

## Consequences

### Positive

- Clear responsibility boundaries between services.
- Objective data remains reliable and traceable.
- AI capabilities can evolve independently.
- Different AI models can be evaluated without changing the domain core.
- Human validation can be integrated naturally.

### Negative

- Requires communication between services.
- The system architecture is more complex than a single AI application.
- Context preparation for AI analysis requires additional processing.
