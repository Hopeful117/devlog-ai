# ADR-001 - DevLog AI as Reference Implementation

## Status

Accepted

## Date

2026-07-20

## Context

DevLog AI aims to provide a methodology and tooling to preserve the technical memory of software projects.

To ensure the relevance and quality of this approach, the project itself should follow the same principles it intends to provide to other projects.

A documentation system that is not applied to its own development would not provide sufficient validation of its concepts.

## Decision

DevLog AI will be developed as its own reference implementation.

The project must apply its own knowledge management principles during its lifecycle.

Major:

- architectural decisions,
- engineering challenges,
- significant evolutions,
- technical choices,

should be captured using the DevLog knowledge model.

The project will serve as the first real-world repository analyzed and documented by DevLog AI.

## Alternatives Considered

### Treat DevLog AI as a normal project

This would allow faster development but would not validate the methodology in a realistic environment.

### Build the documentation system first and apply it later

This would delay feedback from real usage and could lead to designing features without practical validation.

## Consequences

### Positive

- DevLog AI validates its own concepts through real usage.
- Architectural decisions remain traceable.
- The project becomes a demonstration of engineering practices.
- The generated documentation can be used as a portfolio showcase.

### Negative

- Development requires more discipline.
- Documentation becomes part of the development workflow.
- Some decisions may require additional time to formalize.
