# ADR-004 - Human Validation Workflow

## Status

Accepted

## Date

2026-07-20

## Context

DevLog AI uses artificial intelligence to interpret software project evolution and propose structured knowledge.

However, software engineering decisions often depend on context that cannot always be inferred from technical changes alone.

A code modification may have multiple possible explanations:

- architectural improvement,
- temporary workaround,
- experiment,
- technical debt,
- migration step.

Allowing AI-generated interpretations to automatically become project knowledge could introduce incorrect information and reduce trust in the system.

## Decision

DevLog AI will use a human-in-the-loop validation workflow.

AI-generated interpretations are considered proposals and not authoritative knowledge.

The validation lifecycle is:

Raw Activity

↓

Deterministic Analysis

↓

AI Proposal

↓

Human Review

↓

Validated Project Knowledge

The developer remains responsible for confirming important knowledge elements.

Validation applies especially to:

- Engineering Events,
- Engineering Decisions,
- Challenges,
- Documentation Outputs.

## Alternatives Considered

### Fully autonomous AI documentation

Allow AI to automatically create and update project knowledge.

Rejected because:

- AI interpretation may be incorrect,
- technical context may be missing,
- incorrect knowledge could propagate.

### Manual documentation only

Require developers to manually create all project knowledge.

Rejected because:

- creates too much friction,
- does not solve the documentation burden problem,
- loses the benefit of automation.

### Human validation only for final documents

Allow AI interpretations to directly update knowledge and only validate generated outputs.

Rejected because:

- incorrect assumptions would already exist in the knowledge base,
- generated documentation quality depends on the underlying knowledge.

## Consequences

### Positive

- Maintains trust in generated knowledge.
- Keeps the developer involved in important decisions.
- Allows AI assistance without losing human ownership.
- Creates a clear distinction between suggestions and facts.

### Negative

- Requires user interaction.
- Prevents fully autonomous documentation workflows.
- Adds a validation step before knowledge becomes official.
